# Module Review: ["encoding", "decoding", "ops"]

## Executive Summary

**Findings by severity:** 9 critical, 36 important, 35 suggestion (80 total) across encoding, decoding, and ops.

**Top issues**

- [decoding / statistical-validity] **Reversed EV (REV) is hardcoded identically equal to EV, making the documented EV>REV reactivation control statistically meaningless** — `src/neurospatial/decoding/assemblies.py:1096-1099`
- [decoding / statistical-validity] **Explained variance ignores the control period even when control_correlations is provided** — `src/neurospatial/decoding/assemblies.py:1096-1115`
- [decoding / algorithmic-correctness] **reactivation_strength is structurally blind to reactivation magnitude due to double z-scoring** — `src/neurospatial/decoding/assemblies.py:959-972`
- [encoding / statistical-validity] **Rayleigh p-value uses firing rate (Hz) as the test's sample size, making p-values invalid and scale-dependent** — `src/neurospatial/encoding/directional.py:649-662`
- [decoding / silent-failure] **decode_position / log_poisson_likelihood never check encoding_models n_bins against env.n_bins → silent wrong positions** — `src/neurospatial/decoding/posterior.py:365-399`
- [encoding / silent-failure] **NaN headings silently miscounted into direction bin 0, corrupting head-direction tuning curves** — `src/neurospatial/encoding/_directional_binning.py:183-185`
- [ops / numerical-robustness] **resample_field(method="diffuse") propagates a single out-of-source NaN across the whole output, silently zeroing the entire resampled field** — `src/neurospatial/ops/binning.py:785-804`
- [decoding / numerical-robustness] **credible_region silently includes a NaN bin as highest-density in the HPD region** — `src/neurospatial/decoding/estimates.py:342-369`

**Cross-cutting patterns**

- **NaN handling is the dominant failure mode.** The same class of bug — NaN silently miscounted, propagated, or selected as a max/argmax — recurs across all three modules: head-direction binning (encoding), credible_region / confusion_matrix / posterior_mode (decoding), and resample_field / map_probabilities (ops). Most are silent and corrupt scientific output rather than erroring.
- **Silent broad-except fallbacks.** Repeated `except Exception → degrade silently` pattern: geodesic→Euclidean fallbacks (encoding `_field_metrics.py`, decoding `metrics.py`, ops via heading), grid peak detection, basis spectral radius, and KDTree→all-zeros in `map_probabilities`. Classifiers (`is_head_direction_cell`, `has_phase_precession`, `is_object_vector_cell`, `is_spatial_view_cell`) all swallow ValueError and return False, hiding malformed input.
- **API-convention drift.** RNG seeding (`random_state` vs `rng`), `env`-first ordering, keyword-only (`*`) separators, and posterior-summary parameter naming (`method` vs `summary_method`) are each violated in several places per module — plus an unused-mandatory-`env` footgun appearing in encoding, decoding, and ops.
- **Documentation rot from a global rename.** A find/replace corrupted prose ("uncertainty"/"entropy" → the identifier `posterior_entropy`) and left dead method references (`mrl()`, `peak_view_locations()`, `convolve`) across docstrings, QUICKSTART, and API_REFERENCE.

**Recommended order**

1. **Fix the reactivation/assembly statistics first (`assemblies.py`).** Three criticals plus two importants cluster here (REV==EV, control-period ignored, double z-scoring, r vs r²); the entire documented reactivation-control workflow currently produces meaningless results, and the REV bug is untested.
2. **Close the silent-NaN and dimension-mismatch corruptors.** `decode_position` n_bins check, head-direction NaN binning, `resample_field` diffuse NaN, `credible_region`/`confusion_matrix` NaN rows — these silently yield wrong positions/fields with no error, the highest risk for invalid published results. Add the missing NaN-propagation and n_components>rank tests alongside.
3. **Then statistical validity in encoding** (Rayleigh sample-size, phase-precession slope-independent p-value) — invalid p-values affect cell-classification conclusions.
4. **Finally, batch the convention/doc cleanups** (RNG naming, keyword-only separators, `convolve`/`mrl()` dead refs, rename-corrupted prose) — low-risk, high-volume, fixable together.

## `encoding`

### Critical (must fix before use)
- [statistical-validity] **Rayleigh p-value uses firing rate (Hz) as the test's sample size** — `src/neurospatial/encoding/directional.py:649-662`. `rayleigh_pvalue()` passes per-bin Hz as `weights`, so z = sum(Hz)·R² scales with firing-rate magnitude (identical tuning at 50 Hz vs 5 Hz yields ~10× larger z), invalidating HD-cell significance; plumb the already-computed per-bin spike counts into the result and weight the test by counts.
- [silent-failure] **NaN headings silently miscounted into direction bin 0** — `src/neurospatial/encoding/_directional_binning.py:183-185`. `np.digitize(NaN,...)-1` yields `n_bins` and `frame_bins[frame_bins>=n_bins]=0` folds NaN headings (tracking dropouts) into bin 0, corrupting the tuning curve, preferred direction, and MVL; reject non-finite headings in `validate_trajectory` or mask them out of both occupancy and spike counts.

### Important (should fix)
- [numerical-robustness] **`tuning_width` returns NaN when a NaN bin is adjacent to the half-max crossing** — `src/neurospatial/encoding/directional.py:522-547`. The flat-curve guard uses `np.nanmin` so NaN (unvisited) bins survive into the HWHM scan; interpolating across a NaN `break`s the loop and discards a well-defined crossing on the opposite side, biasing population tuning-width stats toward fully-sampled cells; skip/interp across NaN bins and average only the finite half-widths.
- [statistical-validity] **Phase-precession p-value is slope-independent** — `src/neurospatial/encoding/phase_precession.py:318-352`. The slope is fit by maximizing residual concentration over a (−2π,2π) search, but `correlation`/`pval` come from `circular_linear_correlation(phases, positions)`, which ignores the fitted slope and carries no penalty for the search; report significance at the fitted slope (e.g. shuffle positions and re-fit) or document the decoupling.
- [silent-failure] **`is_head_direction_cell` swallows ValueError and returns False** — `src/neurospatial/encoding/directional.py:1884-1895`. `except (ValueError, RuntimeError): return False` converts genuine input errors (mismatched lengths, bad angle_unit) into a silent "not an HD cell" verdict; validate inputs outside the try and let validation errors propagate.
- [silent-failure] **`has_phase_precession` swallows ValueError and returns False** — `src/neurospatial/encoding/phase_precession.py:399-407`. A length-mismatch between positions and phases (classic alignment bug) becomes a silent `False` for every cell; separate input validation from the non-significance fallback and let shape/length errors propagate.
- [api-consistency] **`is_object_vector_cell` free function and `EgocentricRateResult.is_object_vector_cell` use different criteria** — `src/neurospatial/encoding/egocentric.py:1869-1979 and 388-462`. The free function classifies by `object_vector_score`/`min_peak_rate`; the result method by `egocentric_spatial_information`/`min_info`, so the same cell gets different OVC labels by entry point (unlike the HD/view pairs, which delegate); make the free function delegate to the result method or rename one.
- [ux-ergonomics] **`is_object_vector_cell()`/`is_spatial_view_cell()` swallow ValueError/RuntimeError and return False** — `src/neurospatial/encoding/egocentric.py:1955-1979` (and `view.py:1541-1555`). Genuine input errors during population screening are indistinguishable from a real negative classification, silently dropping cells; validate up front and only map the narrow empty-spike-train case to False.
- [performance-memory] **`rate_map_centroid` geodesic path is O(F²) NetworkX Dijkstra calls** — `src/neurospatial/encoding/_field_metrics.py:187-220`. A nested Python loop over field bins runs single-pair `nx.shortest_path_length` on the full graph per pair (O(F²·E log V)), dominating population loops; replace with one `geodesic_distance_matrix(indices=field_bins)` call and a vectorized weighted-medoid argmin.
- [docs-consistency] **QUICKSTART references non-existent `peak_view_locations()`** — `.claude/QUICKSTART.md:564, 583`. The method was renamed to singular `peak_view_location()` (`view.py:249, 625`); copy-pasted code raises AttributeError — update both call sites.
- [docs-consistency] **API_REFERENCE lists `DirectionalRateResult.mrl()` which does not exist** — `.claude/API_REFERENCE.md:346`. The actual method is `mean_vector_length()` (`directional.py:380`); calling `.mrl()` raises AttributeError — fix to `.mean_vector_length()` (and the batch plural).
- [docs-consistency] **QUICKSTART OVC block calls single-neuron methods on a batch result and mis-unpacks `plot()`** — `.claude/QUICKSTART.md:459-468`. `result` is a batch `EgocentricRatesResult` but the block calls `is_object_vector_cell()`/`preferred_distance()` (single-only) and `fig, ax = result.plot()` (batch `plot()` needs `idx`; single `plot()` returns one Axes); mirror the view-cell section with an explicit `single` variable and `detect_ovcs()`/`preferred_distances()`.

### Suggestions (nice to have)
- [numerical-robustness] **`spatial_information` divides by `np.log(base)` with no guard for base<=0 or base==1** — `src/neurospatial/encoding/_core_numpy.py:383`. `base=1` gives division by zero (inf/NaN) not caught by the final mask, and `base<=0` yields NaN; validate `base` in the public boundary functions (`spatial_information`, `batch_spatial_information`, `information_per_second`, `mutual_information`).
- [statistical-validity] **No multiple-comparison handling in batch HD/view/OVC classification** — `src/neurospatial/encoding/directional.py:1155-1196` (and `view.py:744-750`, `egocentric.py:828`). Per-neuron alpha=0.05 across a population guarantees false positives; offer optional FDR/Bonferroni or a shuffle null and document that the default thresholding is uncorrected.
- [units-and-frames] **`gaussian_kde` on the egocentric polar env mixes cm and radians in one scalar bandwidth** — `src/neurospatial/encoding/egocentric.py:1098-1108`. The kernel takes Euclidean distance over (cm, rad) bin centers and divides by one bandwidth, so `bandwidth=5` means 5 cm radially but 5 rad (~286°) angularly; restrict to `binned`, normalize the angle axis, or document the mismatch.
- [reproducibility-rng] **Docstring falsely calls deterministic grid peak detection "stochastic"** — `src/neurospatial/encoding/grid.py:1280-1286`. `find_peaks`/`peak_local_max` have no RNG; the <1% differences come from independently recomputed autocorrelation inputs, not randomness — reword to avoid implying nondeterminism and a nonexistent seed.
- [tests] **No end-to-end HD-cell recovery test for `compute_directional_rate`** — `tests/encoding/test_encoding_directional.py:2740-2906`. Integration tests use only random headings/spikes (shape, non-negativity, rad/deg equivalence); the one preferred-direction test bypasses binning/occupancy via the dataclass, so a heading-binning or occupancy-normalization error would pass silently — add a simulated-HD-cell test asserting recovered preferred direction, mirroring `compute_spatial_rate`.
- [silent-failure] **Broad `except Exception` in grid peak detection silently returns no peaks** — `src/neurospatial/encoding/grid.py:1047-1057`. Unexpected programming errors become "no peaks" → NaN grid score, masking real defects; catch only the expected degenerate-input exception type.
- [silent-failure] **Geodesic field distance silently falls back to Euclidean on any exception** — `src/neurospatial/encoding/_field_metrics.py:900-914`. A broad `except Exception` substitutes a different metric than requested after only a UserWarning; narrow the catch (NetworkXNoPath/NetworkXError) and return NaN or make the fallback opt-in.
- [api-consistency] **`view_distance`/`gaze_model` parameter order reversed between `compute_view_rate` and `is_spatial_view_cell`** — `src/neurospatial/encoding/view.py:867-869 and 1474-1476`. Sibling gaze functions declare these keyword-only params in opposite order; align them (gaze_model before view_distance).
- [api-consistency] **Batch result classes use inconsistent names for the population classifier** — `src/neurospatial/encoding/spatial.py:1042`. `SpatialRatesResult.classify` vs `detect_hd_cells`/`detect_ovcs`/`detect_view_cells` across sibling classes hurts discoverability; adopt one convention or add shared aliases.
- [ux-ergonomics] **`is_object_vector_cell()` requires a non-optional `env` the default euclidean metric never uses** — `src/neurospatial/encoding/egocentric.py:1869-1883`. The underlying compute fn already accepts `env=None`; change the signature to `env: Environment | None = None` to match and avoid forcing a throwaway Environment.
- [ux-ergonomics] **`compute_egocentric_rate/rates` force a leading positional `None` for the common euclidean case** — `src/neurospatial/encoding/egocentric.py:1037-1051`. `env` is the first positional param with no default though euclidean ignores it; give it `= None` so euclidean callers can omit it.
- [ux-ergonomics] **`compute_spatial_rate` `min_occupancy` default (0.0) disagrees with the documented recommended 0.5** — `src/neurospatial/encoding/spatial.py:1257`. The minimal call keeps barely-visited bins, producing noisier rate maps than the CLAUDE.md guidance; align the default or warn in the docstring that 0.0 keeps all visited bins.
- [performance-memory] **`gaussian_kde` smoothing materializes a dense (n_bins × n_bins) weight matrix** — `src/neurospatial/encoding/_smoothing.py:76-105`. At 10k bins this is ~800 MB and can OOM; truncate the kernel to a sparse neighborhood or emit a ResourceWarning steering large grids to `diffusion_kde`.

### Strengths
- [algorithmic-correctness] No correctness defects survived verification (0 found) — the core encoding algorithms (binning math, rate computation, autocorrelation/grid metrics) are sound.
- The encoding module is broadly solid: the surviving issues cluster in error-handling ergonomics (broad excepts), doc drift, and statistical-test sample-size conventions rather than in the numerical core. Spatial-rate computation in particular has a genuine end-to-end recovery test, and most numerical paths are robust apart from a couple of unguarded edge cases (base validation, NaN-adjacent crossings).

## `decoding`

### Critical (must fix before use)
- [algorithmic-correctness] **reactivation_strength is structurally blind to reactivation magnitude due to double z-scoring** — `src/neurospatial/decoding/assemblies.py:959-972`. `assembly_activation` unconditionally z-scores each period to unit variance, so `mean(|act_match|)/mean(|act_template|)` is ~1 by construction regardless of true activation; add a `z_score_output=False` path (or use squared raw projections) and test that genuinely larger match activity yields strength > 1.
- [statistical-validity] **Reversed EV (REV) is hardcoded equal to EV, making the EV>REV reactivation control meaningless** — `src/neurospatial/decoding/assemblies.py:1096-1099`. `rev = ev` means the documented "EV > REV: forward reactivation" can never hold; implement REV as the role-swapped partial correlation (Kudrimoti) so it genuinely differs from EV.
- [statistical-validity] **Explained variance ignores control_correlations even when provided** — `src/neurospatial/decoding/assemblies.py:1096-1115`. `ev = r_tm**2` is set before the control block and never updated, so the baseline is never partialled out of the headline EV; set `ev = partial_corr**2` when a control is supplied.
- [numerical-robustness] **credible_region silently includes NaN bin as highest-density in HPD region** — `src/neurospatial/decoding/estimates.py:342-369`. `argsort(...)[:, ::-1]` ranks NaN bins first and cumsum-over-NaN fills the row with all bins; detect NaN rows up front (raise or return empty) or mask NaN to -inf before argsort.
- [silent-failure] **decode_position / log_poisson_likelihood never check encoding_models bin count against env.n_bins → silent wrong positions** — `src/neurospatial/decoding/posterior.py:365-399`. A bin-axis mismatch passes through the neuron-axis matmul and yields plausible-but-wrong coordinates; assert `encoding_models.shape[1] == env.n_bins` (and `spike_counts.shape[1] == encoding_models.shape[0]`) with a clear ValueError.

### Important (should fix)
- [numerical-robustness] **confusion_matrix mis-handles NaN posterior rows (miscount in 'map', NaN poisoning in 'expected')** — `src/neurospatial/decoding/metrics.py:338-348`. All-NaN rows are tallied as bin 0 (map) or added directly into the matrix (expected); validate finiteness and either raise or exclude NaN rows. (also [tests] — no regression test feeds confusion_matrix a NaN-containing posterior at `tests/decoding/test_metrics.py:368-657`.)
- [statistical-validity] **partial_correlation returned un-squared while explained_variance is squared (r vs r² scale mix)** — `src/neurospatial/decoding/assemblies.py:1113-1118`. The two reactivation numbers on one result object live on incompatible scales; document the distinction or expose a squared partial-EV field, and fix the no-control fallback for consistency.
- [units-and-frames] **Slope-to-speed conversion references nonexistent `env.bin_size` and conflates bin area with width** — `src/neurospatial/decoding/trajectory.py:135, 424`. `result.slope * env.bin_size` hits AttributeError, and the real `env.bin_sizes` is an area; use a true linear bin width (e.g. `np.median(np.diff(env.bin_centers[:, 0]))`).
- [units-and-frames] **`slope * bin_width` is meaningful only for 1D tracks but documented "for regular grids"** — `src/neurospatial/decoding/trajectory.py:419-424`. Flat bin index walks a 2D grid row-major, so the conversion is dimensionless nonsense there; scope guidance to `env.is_linearized_track` and drop the misleading comment.
- [silent-failure] **log_poisson_likelihood / poisson_likelihood silently accept 1D spike_counts and collapse the time axis** — `src/neurospatial/decoding/likelihood.py:118-181`. A `(n_neurons,)` input returns `(n_bins,)` via broadcasting; raise if `spike_counts.ndim != 2` (and check `shape[1] == encoding_models.shape[0]`).
- [silent-failure] **decode_position stores a `times` array of mismatched length without validation** — `src/neurospatial/decoding/posterior.py:394-399`. A wrong-length `times` mis-aligns the posterior time axis in `plot()` and only errors later in `to_dataframe()`; raise if `len(times) != posterior.shape[0]`. (also [ux-ergonomics] — same gap, inconsistent with the rich validation already present.)
- [silent-failure] **decoding_error(metric='geodesic') silently falls back to Euclidean for out-of-environment positions** — `src/neurospatial/decoding/metrics.py:162-172`. Out-of-env bins get Euclidean values mixed into the geodesic output with no warning, biasing reported error; warn with a count, document it, or add an `outside=` policy.
- [tests] **detect_assemblies crashes with cryptic IndexError when n_time_bins < n_components ≤ n_neurons** — `tests/decoding/test_assemblies.py:259-265`. The SVD rank is exceeded for short recordings (an explicitly anticipated case); cap `n_components` at `min(n_neurons, n_time_bins)` with a clear ValueError and test the boundary.
- [api-consistency] **detect_assemblies uses `random_state` while every other RNG-seeded API uses `rng`** — `src/neurospatial/decoding/assemblies.py:374`. Even its sibling `fit_linear_trajectory` uses `rng: Generator | int | None`; rename to `rng`, resolve via `_ensure_rng`, and pass a seed down to sklearn internally.
- [api-consistency] **Same {'map','expected'} choice is `summary_method` in confusion_matrix but `method` in fit_isotonic_trajectory** — `src/neurospatial/decoding/metrics.py:233`. `method` is also overloaded for unrelated algorithm choices; standardize on `summary_method` for the bin-position summary across siblings.
- [ux-ergonomics] **`spike_counts` has transposed shapes across the subpackage — silent misuse footgun** — `src/neurospatial/decoding/assemblies.py:384, 760, 863`. decode_position expects `(n_time_bins, n_neurons)` while the assembly functions expect `(n_neurons, n_time_bins)`, so a transposed array silently returns garbage; cross-reference the opposite convention prominently and add a transpose heuristic warning.
- [docs-consistency] **Docstring prose corrupted by global rename: "uncertainty" replaced with identifier "posterior_entropy"** — `src/neurospatial/decoding/trajectory.py:12, 102, 373, 386, 390, 392`. `fit_linear_trajectory` quantifies Monte-Carlo slope uncertainty, not posterior entropy; restore "uncertainty" in prose (also `decoding/__init__.py:59`).
- [docs-consistency] **Self-referential corrupted prose "Measures the posterior_entropy in the position estimate"** — `src/neurospatial/decoding/estimates.py:203, 205`. Entropy is described as measuring "the posterior_entropy"; replace with "uncertainty" (same fix in `_result.py:175, 200, 361`).
- [docs-consistency] **REV docs contradict implementation: "EV should exceed REV" is impossible since rev == ev** — `src/neurospatial/decoding/assemblies.py:1098-1099`. Docstrings present REV as an independent control; either make REV genuinely asymmetric or document that REV==EV and the real control is `partial_correlation`.
- [performance-memory] **fit_linear_trajectory loops in Python over n_samples (default 1000) recomputing loop-invariant regression terms** — `src/neurospatial/decoding/trajectory.py:513-522`. The per-sample OLS vectorizes exactly across the sample axis; compute the x-terms once and solve all samples with one matrix-vector product (numerically identical, including the denom==0 fallback).

## `ops`

### Critical (must fix before use)
- [numerical-robustness] **resample_field(method="diffuse") propagates a single out-of-source NaN across the whole output** — `src/neurospatial/ops/binning.py:785-804`. A dense diffusion kernel multiplies every output bin by every input bin, so one out-of-source NaN annihilates the entire field; zero-fill the outside-source NaNs before `apply_kernel`, then re-impose NaN only on genuinely-outside bins afterward.
- [silent-failure] **heading_from_velocity divides by dt with no validation; dt<=0 or unsorted times silently yields NaN or 180°-wrong headings** — `src/neurospatial/ops/egocentric.py:635-736` (also flagged [numerical-robustness] at 708-736). dt==0 returns all-NaN headings (the low-speed warning never fires); dt<0 rotates every heading by π. Add `if not np.isfinite(dt) or dt <= 0: raise ValueError(...)` after the length check.
- [tests] **Inverse-distance-weighted mode of map_probabilities is completely untested** — `src/neurospatial/ops/alignment.py:388-453`. The entire IDW weighting/normalization/clamp path and `n_neighbors` validation are never exercised (all tests use `mode="nearest"`), so a wrong weighting would silently produce a plausible-but-wrong remapped distribution; add tests for hand-computed splits, mass conservation, the `n_neighbors` clamp, and the `k_eff==1` branch.

### Important (should fix)
- [algorithmic-correctness] **estimate_transform rigid/similarity reflection-correction produces a suboptimal rotation (classic Kabsch bug)** — `src/neurospatial/ops/transforms.py:1269-1276`. Negating the last column of `R` when `det(R)<0` does not yield the optimal proper rotation; use the SVD det-sign trick (`D=eye; D[-1,-1]=sign(det(U@Vt)); R=U@D@Vt`) and add a reflected-point regression test.
- [units-and-frames] **compute_viewshed silently caps ray distance at a hardcoded 200 spatial units** — `src/neurospatial/ops/visibility.py:830`. The undocumented, non-parameterized cap silently drops distant-but-visible bins for large/mm-unit environments (and is inconsistent with siblings that expose `max_distance`); promote it to a keyword param defaulted from env extent and document its units.
- [silent-failure] **map_probabilities silently returns an all-zero probability array on any KDTree failure** — `src/neurospatial/ops/alignment.py:374-382, 424-432, 578-586`. Broad `except Exception` + warn + return zeros yields a wrong-but-plausible flat/empty distribution and masks real shape/scipy bugs; narrow the catch and raise instead of warning for unexpected failures.
- [api-consistency] **basis.py public functions lack the keyword-only `*` separator before algorithm params** — `src/neurospatial/ops/basis.py:117-122, 347-355, 575-583, 760-768, 1005-1010`.
