**Top issues**

- [annotation / silent-failure] **Alpha-shape boundary inference silently returns a non-Polygon / empty geometry, producing a degenerate boundary** — `src/neurospatial/annotation/_boundary_inference.py:289-310`
- [annotation / tests] **No regression test for edge_order/edge_spacing override staleness after delete_node/delete_edge — silently wrong linearization** — `src/neurospatial/annotation/_track_state.py:205-249`
- [animation / docs-consistency] **QUICKSTART/overlay docs use removed `PositionOverlay(data=...)` and raw-list `skeleton=` kwargs — examples raise TypeError/AttributeError** — `.claude/QUICKSTART.md:674-693`
- [animation / performance-memory] **Default reuse-artists video path never renders events, so events silently vanish from exported video** — `src/neurospatial/animation/_parallel.py:1066-1104`

**Cross-cutting patterns**

- **Silent acceptance of degenerate/invalid numeric input** recurs in all three modules: alpha-shape non-Polygon (annotation), NaN/Inf and non-ascending timestamps in spike generation and width=0 fields (simulation), NaN data-range warnings and integer-dtype truncation in overlays (animation). Several violate explicit "warn, never raise" or documented-behavior contracts.
- **Circular quantities interpolated/handled linearly:** HeadDirectionOverlay interpolates heading across the ±π wrap (animation, two findings); object-vector angle-convention mixing (simulation).
- **Stale/broken docs against current API:** removed overlay kwargs (`data=`, `skeleton=[...]`, `skeleton_color/width`), non-existent imports (`PositionOverlay`, `Role`), and wrong default values (multiple `baseline_rate`, `border_width`, `boundary_config`) span animation, annotation, and simulation — a systemic docstring/doc-drift problem.
- **Keyword-only (`*`) convention violated** for algorithm params across annotation (`method`, `simplify_tolerance`) and simulation (`headings`, sibling trajectory functions), contradicting the canonical argument order in CLAUDE.md.
- **RNG stream recoupling** in simulation example sessions (grid_cell, boundary_cell) reintroduces the trajectory/spike correlation that `simulate_session` was fixed to avoid.

**Recommended order**

1. **Scientific-correctness criticals first** — the anisotropic-width double-normalization and width=0 NaN in PlaceCellModel, plus non-ascending/NaN timestamp handling in `generate_poisson_spikes`. These silently corrupt simulated ground truth that downstream encoding/decoding tests rely on, so every dependent result is suspect until fixed.
2. **Silent geometry/coordinate failures** — alpha-shape non-Polygon degenerate boundaries (annotation) and integer-dtype overlay truncation (animation), since both produce plausible-looking-but-wrong spatial output with no error.
3. **Doc/API-consistency batch** — fix the removed-kwarg overlay examples, non-existent imports (`PositionOverlay`, `Role`), and wrong documented defaults together; they are low-risk, high-frequency user-facing breakages and several share the same root drift.
4. **Test-coverage gaps** (edge_order staleness, sinusoidal trajectory, event-render-in-video) to lock in the above fixes and catch the silent-linearization regression.
5. Remaining suggestions (RNG recoupling, keyword-only ordering, performance loops) as follow-up hygiene.

## `simulation`

### Critical (must fix before use)
- [algorithmic-correctness / statistical-validity] **Anisotropic place field divides by width twice, producing a near-flat (wrong) Gaussian** — `src/neurospatial/simulation/models/place_cells.py:316-335`. The anisotropic branch sets `distances` to the already width-normalized Mahalanobis distance, then line 334 divides by `mean(width)` again (and the line-330 clip mixes sigma-units with a physical length); fix by keeping the anisotropic distance in sigma units and evaluating `exp(-0.5 * d_sigma**2)` with a unit divisor (numerically: width=[5,5] should equal scalar width=5.0). Add a regression test asserting 1-sigma offset gives `exp(-0.5)`.
- [numerical-robustness] **PlaceCellModel.firing_rate yields NaN at field center when width=0, silently producing zero spikes there** — `src/neurospatial/simulation/models/place_cells.py:193-201,321,334`. `__init__` never validates width>0 (unlike every sibling model), so width=0 gives 0/0→NaN that propagates to zero spikes at the peak with no warning; reject non-positive width at construction (`if np.any(np.asarray(self.width) <= 0): raise ValueError(...)`).
- [silent-failure] **generate_poisson_spikes silently produces wrong spike trains for non-uniform / non-ascending timestamps** — `src/neurospatial/simulation/spikes.py:128-147`. A single `dt = times[1]-times[0]` is applied to every bin; non-uniform times give wrong spike counts and descending times collapse probabilities to ~0, all silently, and no length check exists between `firing_rate` and `times`. Validate equal length, strictly increasing times, and uniform dt (or use per-bin `np.diff`).

### Strengths
- **Reproducibility at the session level is solid.** `simulate_session` correctly spawns three independent RNG sub-streams via `SeedSequence(seed).spawn(3)` and has a dedicated regression test guarding trajectory/spike stream independence; the surviving RNG findings are convenience-session functions that bypass this otherwise-correct pattern.
- **Sibling tuning models validate their widths/concentrations.** GridCell, BoundaryCell, HeadDirection, ObjectVector, and SpatialView models all reject non-positive tuning parameters at construction — only PlaceCellModel's width slipped through.
- **Distance/spike-generation math is otherwise sound.** The isotropic place-field path, per-bin probability via `1 - exp(-rate*dt)`, and the population dispatch are correct; the confirmed numerical issues are concentrated in unvalidated edge inputs (NaN/Inf, zero/anisotropic width, degenerate time arrays) rather than the core algorithms.
