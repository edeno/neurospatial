# Module Review: ["io", "layout", "regions"]

## Executive Summary

**Findings by severity:** 6 critical, 35 important, 36 suggestion (77 total) across io, layout, regions.

**Top issues**

- [layout / units-and-frames] **ImageMaskLayout uses (y,x) for bin_centers/grid_edges/point_to_bin but (x,y) for dimension_ranges, breaking the cross-engine coordinate contract and producing wrong bin lookups for non-square pixels** — `src/neurospatial/layout/engines/image_mask.py:130-157`
- [io / api-consistency] **NWB write_environment/read_environment silently drops coordinate_kind, flipping polar environments to Cartesian on round-trip** — `src/neurospatial/io/nwb/_environment.py:230-243, 517-577`
- [regions / silent-failure] **_rle_to_mask silently produces a wrong mask on out-of-bounds/negative RLE indices; docstring promises a ValueError that never fires** — `src/neurospatial/regions/io.py:212-250`
- [layout / silent-failure] **MaskedGridLayout.build does not validate active_mask dtype, allowing silent corruption from integer/float masks** — `src/neurospatial/layout/engines/masked_grid.py:89-119`
- [io / tests] **No to_file/from_file round-trip test for Graph or Polygon layouts — both crash on save (untested public-API failure)** — `tests/io_tests/test_io.py:32-133`
- [io / numerical-robustness] **timestamps_from_series emits NaN/Inf timestamps when rate <= 0, silently poisoning downstream encoding/decoding** — `src/neurospatial/io/nwb/_adapters.py:57-70`
- [regions / units-and-frames] **point_tolerance default of 1e-8 is a float-equality epsilon applied to physical cm/pixel coordinates, so point-region membership silently returns all-False for real positions** — `src/neurospatial/regions/ops.py:192-246`

**Cross-cutting patterns**

- **Silent corruption over loud failure.** Recurs in every module: NWB drops coordinate_kind and unit attributes; RLE decode truncates bad runs; MaskedGrid skips dtype checks; layout helpers return all -1 on dimension mismatch; read_position/read_pose/read_head_direction skip length-agreement checks. Bad data flows downstream as plausible-but-wrong results.
- **Coordinate-frame / axis-order contract violations.** ImageMaskLayout (y,x vs x,y), NWB head-direction angle convention and degree/radian units, polar coordinate_kind loss — multiple places silently disagree on frame or axis order.
- **Docstrings contradict implementations.** ~15 findings: region_center's `None` return, plot_regions signature, write_environment exception type, read_environment Hexagonal claim, write_trials nonexistent `write_intervals()`, plus several doc-site/`.claude` examples using renamed or nonexistent kwargs.
- **Untested failure and validation branches.** Public validators, schema-mismatch warnings, RLE bounds, and layout round-trips have zero coverage — which is why several of the criticals (Graph/Polygon save crash, coordinate_kind loss) went unnoticed.

**Recommended order**

1. **Correctness/silent-corruption criticals first**, since they produce wrong numbers with no error: ImageMask axis-order, NWB coordinate_kind round-trip, RLE bounds handling, MaskedGrid dtype validation. Pair each fix with the missing regression test (Graph/Polygon round-trip, polar round-trip, RLE bounds).
2. **Numerical-robustness and silent-failure important items** that poison downstream analyses: NaN/Inf timestamps, length-agreement checks in read_position/read_pose/read_head_direction, head-direction unit/convention enforcement, int32 overflow in get_n_bins, point_tolerance default.
3. **Docs/API/UX consistency last** — high volume but low blast radius; batch the docstring-vs-signature and renamed-kwarg fixes (plot_regions, region_center, write_trials, the `.claude`/docs examples) together once behavior is correct.

Fix correctness-with-no-error issues before anything cosmetic: they are the ones silently corrupting scientific results, and the missing tests are why they survived.

## `io`

### Critical (must fix before use)
- [algorithmic-correctness / api-consistency] **NWB round-trip silently drops coordinate_kind, flipping polar environments to Cartesian** — `src/neurospatial/io/nwb/_environment.py:230-243, 517-591`. `write_environment` never serializes `env.coordinate_kind` and `read_environment` never restores it, so a `from_polar_egocentric` env (`coordinate_kind='polar'`) silently becomes Cartesian on reload, producing wrong egocentric/polar results with no error; persist `coordinate_kind` in the metadata JSON and restore it on read (mirroring the already-fixed `files.py` path).
- [tests] **No to_file/from_file round-trip test for Graph or Polygon layouts — both crash on save** — `tests/io_tests/test_io.py:32-133`. Every persistence test uses RegularGrid; `to_file()` actually raises `TypeError: Object of type Graph/Polygon is not JSON serializable` because `files.py:222` serializes `env.layout_parameters` verbatim, so a hard crash on core public API is untested. Add a parametrized round-trip test over all layout factories (Graph, Polygon, Hexagonal, Masked, ImageMask, TriangularMesh, 3D) and make `layout_parameters` JSON-safe.

### Important (should fix)
- [numerical-robustness] **timestamps_from_series produces NaN/Inf timestamps when rate <= 0** — `src/neurospatial/io/nwb/_adapters.py:57-70`. `rate` is only checked for `None`; a `rate` of 0 yields `[nan, inf, ...]` and a negative rate a decreasing/negative axis, silently poisoning downstream occupancy/dt/decoding. After `rate = float(rate)`, raise if `not np.isfinite(rate) or rate <= 0`.
- [units-and-frames] **read_head_direction does not document or enforce the allocentric (0=East) angle convention** — `src/neurospatial/io/nwb/_behavior.py:242-301`. Returned `angles` are documented only as "radians" with no frame/range, but downstream egocentric/view consumers require allocentric 0=East; NWB CompassDirection imposes no standard frame, so geographic-bearing files are silently misinterpreted. Document the 0=East, +π/2=North contract and put the conversion burden on the caller.
- [units-and-frames] **read_head_direction ignores the SpatialSeries unit attribute (degrees vs radians)** — `src/neurospatial/io/nwb/_behavior.py:297-301`. Line 298 reads `data` as radians without inspecting `spatial_series.unit`; a degrees-encoded file returns values ~57× too large with no warning, unlike `read_position` which consults units. Read `unit` and convert via `np.deg2rad` or warn/raise when not radians-like.
- [units-and-frames] **read_head_direction .ravel() silently corrupts vector-form CompassDirection data** — `src/neurospatial/io/nwb/_behavior.py:298`. Unconditional `.ravel()` flattens legitimate `(n_samples, 2)` heading unit vectors into interleaved scalars of length `2*n_samples`, mismatching timestamps and yielding nonsense angles. Detect dimensionality: ravel only 1D/(n,1); for `(n,2)` raise or convert via `np.arctan2(y, x)`.
- [tests] **files.py round-trip never tested for active_mask / grid_edges / is_linearized_track preservation** — `tests/io_tests/test_io.py:60-113`. Dedicated serialization branches for these fields are only exercised on RegularGrid (where they're trivial), so a regression silently densifying a mask or dropping the linearized-track flag would not fail any test. Assert `active_mask`, `grid_edges`, and `is_linearized_track`/`to_linear()` survive for Masked/ImageMask/Graph envs.
- [tests] **NWB write/read does not preserve coordinate_kind for polar envs, and no test covers it** — `src/neurospatial/io/nwb/_environment.py:230-243`. Same hazard as the critical finding from the test angle: no `tests/nwb/` test round-trips a polar environment. Add a test asserting `loaded.coordinate_kind == 'polar'` / `loaded.is_polar` after an NWB round-trip (it should fail today and drive the fix).
- [silent-failure] **read_position / read_head_direction return positions and timestamps with no length-agreement check** — `src/neurospatial/io/nwb/_behavior.py:85-88, 298-299`. The explicit-timestamps branch returns timestamps verbatim with no `len(data) == len(timestamps)` check, so a truncated/mis-clocked file silently yields mismatched arrays that produce a plausible-but-wrong place field. Raise a `ValueError` when lengths disagree.
- [silent-failure] **read_pose assumes all bodyparts share the first series' timestamps without verifying length** — `src/neurospatial/io/nwb/_pose.py:82-96`. The docstring promises shared timestamps but the code takes them from the first series only ("they should all be the same", unverified), silently misaligning every other bodypart against the wrong time axis. Verify each subsequent series' length (and ideally values) matches the reference and raise on mismatch.
- [ux-ergonomics] **write_trials points users to a nonexistent write_intervals() escape hatch** — `src/neurospatial/io/nwb/_events.py:608, 780`. Both the docstring and the `NotImplementedError` message direct users to `write_intervals()`, which does not exist or get exported anywhere, leaving users stuck at the overwrite wall. Implement/export `write_intervals()` or replace both references with a real call.
- [ux-ergonomics] **write_environment docstring documents wrong exception type for unfitted Environment** — `src/neurospatial/io/nwb/_environment.py:147-150`. Docstring says `ValueError` for the unfitted case, but the code raises `EnvironmentNotFittedError` (a `RuntimeError` subclass), so `except ValueError` will not catch it. Split the Raises section: document `EnvironmentNotFittedError` for unfitted and keep `ValueError` for the duplicate-name case.
- [docs-consistency] **ADVANCED.md uses non-existent `events_name` keyword for read_events()** — `.claude/ADVANCED.md:50`. `read_events(nwbfile, events_name="licks")` raises `TypeError`; the real signature is `read_events(nwbfile, table_name, ...)`. Change the example to `read_events(nwbfile, "licks")`.
- [docs-consistency] **read_environment docstring wrongly lists Hexagonal as fully grid-reconstructed** — `src/neurospatial/io/nwb/_environment.py:476-482`. Hexagonal has an empty `grid_edges` tuple so `_extract_grid_data` returns None and it goes through the KDTree fallback, not MaskedGrid reconstruction; the docstring overstates `point_to_bin_index` fidelity. Move Hexagonal into the non-grid/KDTree-fallback list.
- [performance-memory] **write_environment replicates JSON metadata strings total_grid_cells times, blowing up on-disk size for sparse/fine grids** — `src/neurospatial/io/nwb/_environment.py:245-291`. All columns (including three identical JSON blobs for regions/metadata/grid_data) are padded to the FULL dense `total_grid_cells` instead of active `n_bins`, so a 1000×1000 grid writes ~0.5 GB per string column even though `read_environment` only reads index 0. Store the dense `active_mask` separately and the scalar JSON blobs once (length-1), decoupling them from the dense grid count.

### Suggestions (nice to have)
- [numerical-robustness] **Explicit NWB timestamps passed through without finiteness validation** — `src/neurospatial/io/nwb/_adapters.py:53-55`. The explicit-timestamps branch returns the array verbatim with no `np.isfinite` / monotonicity check, unlike the write side; corrupt time axes silently enter encoding/decoding. Warn or raise when `not np.all(np.isfinite(ts))`.
- [numerical-robustness] **_ReconstructedLayout.bin_sizes can return zero/over-amplified bin volumes for duplicate bin_centers** — `src/neurospatial/io/nwb/_environment.py:788-810`. `median_spacing ** n_dims` becomes 0 when most reconstructed centers coincide, poisoning downstream rate normalization via div-by-zero; only the `<2`-point case is guarded. Fall back to `np.ones(...)` (and/or warn) when `median_spacing <= 0` or non-finite.
- [tests] **Schema-version-mismatch warning branches in files.py and NWB read are untested** — `tests/io_tests/test_io.py:13-133`. The warn-and-continue paths in `from_file`/`from_dict`/`read_environment` on schema drift are never exercised. Mutate `schema_version` to a fake value and assert `pytest.warns(UserWarning, match='Schema version mismatch')` while still returning a usable Environment.
- [tests] **1D-array validation branch in write_laps/write_region_crossings is untested** — `tests/nwb/test_events.py:504-910`. The `must be 1D array` guards are never hit by the suite. Add a test passing a 2D array and assert `pytest.raises(ValueError, match='must be 1D')`.
- [silent-failure] **timestamps_from_series does not validate explicit timestamps length against data** — `src/neurospatial/io/nwb/_adapters.py:53-55`. Centralizing a `ts.shape[0] != len(series.data)` check here would protect every caller (read_position/head_direction/pose) in one place; the function already accesses `series.data`. Raise on mismatch before returning.
- [silent-failure] **_get_position_units silently defaults to 'cm' with no signal** — `src/neurospatial/io/nwb/_environment.py:1152-1153`. A unitless/pixel SpatialSeries silently stamps `units='cm'`, masking unit mismatches that propagate into bandwidths/distances. Emit a `logger.warning`/`warnings.warn` when the fallback is used.
- [silent-failure] **from_file / from_dict warn-and-continue on schema mismatch can silently misread fields** — `src/neurospatial/io/files.py:354-361, 558-566`. An easily-swallowed `UserWarning` is a weak guard against reconstructing a wrong-but-plausible Environment from a drifted format. Hard-fail (raise) on a differing major schema version while keeping warn-and-continue for minor drift.
- [api-consistency] **Inconsistent placement of the `description` parameter across sibling NWB writers** — `src/neurospatial/io/nwb/_events.py:211-217, 419-422, 1004-1008`. `write_laps`/`write_region_crossings`/`write_place_field`/`write_occupancy` put `description` before `*` (positional-or-keyword) while `write_events`/`write_trials` make it keyword-only, so users can't call them interchangeably. Standardize on keyword-only after `*` per project convention.
- [ux-ergonomics] **environment_from_position silently defaults units to 'cm' without disclosing it** — `src/neurospatial/io/nwb/_environment.py:1026-1028, 1152-1153`. The public `units` docstring says "auto-detected" but omits the unconditional `'cm'` fallback when the SpatialSeries has no unit. Document the full behavior and optionally warn when the fallback fires.
- [ux-ergonomics] **write_place_field default name='place_field' invites silent collisions across cells** — `src/neurospatial/io/nwb/_fields.py:150-154`. The default name makes the common per-cell loop raise "already exists" on the second cell (or clobber with overwrite=True). Make `name` required or document that per-cell writes need a unique name.
- [ux-ergonomics] **Unused module-level constants in _environment.py reduce clarity** — `src/neurospatial/io/nwb/_environment.py:80-86`. `DEFAULT_PROCESSING_MODULE`, `DEFAULT_ANALYSIS_MODULE`, and `DEFAULT_STATIC_TIMESTAMP` are never referenced (write_environment uses `scratch/`), misleading readers about the serialization contract. Remove them or wire them in.
- [docs-consistency] **io/files.py module docstring describes a directory/zip schema that is not produced** — `src/neurospatial/io/files.py:7-19`. The header claims a directory/zip of four files (metadata.json, arrays.npz, graph.json, regions.json), but `to_file` writes exactly two flat files (`{path}.json` + `{path}.npz`) with graph/regions embedded in the JSON. Update the docstring to the actual two-file output.
- [docs-consistency] **TROUBLESHOOTING error string does not match actual KeyError message** — `.claude/TROUBLESHOOTING.md:403`. The heading `KeyError: No Position found` does not match the real message `No Position data found in NWB file. Searched: [...]`, so users grepping their traceback won't find it. Update the text to match the real message.

### Strengths
- [statistical-validity] No statistical-validity issues surfaced — the io module is pure serialization/IO and correctly contains no statistical estimation that could be invalid.
- [reproducibility-rng] No RNG/reproducibility issues — io performs no randomized operations, so round-trips are deterministic by construction.
- The event/field write paths (`_events.py`) already enforce finiteness and non-negativity validation before writing, which is why the surviving numerical-robustness gaps are concentrated on the comparatively unguarded read side rather than the write side.

## `layout`

### Critical (must fix before use)
- [algorithmic-correctness, units-and-frames] **ImageMaskLayout axis-order mismatch: bin_centers/grid_edges are (y, x) but dimension_ranges is (x, y)** — `src/neurospatial/layout/engines/image_mask.py:130-157`. `grid_edges=(y_edges,x_edges)` and `bin_centers` stacked as `(yv,xv)` make column 0=y, but `dimension_ranges=((x_range),(y_range))` makes dim0=x, breaking the cross-engine contract; for non-square pixels `point_to_bin_index` silently returns wrong bins and plots transpose. Fix: build `grid_edges`/`bin_centers` in (x, y) order to match the other grid engines (or set `dimension_ranges` to (y, x) and document the (row, col) requirement).
- [silent-failure] **MaskedGridLayout.build does not validate active_mask is boolean** — `src/neurospatial/layout/engines/masked_grid.py:89-119`. A 0/1 int or float mask triggers NumPy fancy indexing in `bin_centers = full_grid_bin_centers[mask.ravel()]`, silently producing wrong, inconsistent coordinates (16 vs 4 nodes) with no error. Fix: raise `ValueError` if `active_mask` dtype is not `np.bool_`, matching the check ImageMaskLayout already performs.

### Important (should fix)
- [algorithmic-correctness] **Boundary point active in occupancy (histogramdd) but maps to -1 (digitize)** — `src/neurospatial/layout/helpers/regular_grid.py:227-228, 654-662`. A sample on the exact max edge is counted active by `histogramdd` but `digitize` maps it to bin -1, so a real sample contributes to the active mask yet is unbinnable at query time. Fix: clamp points on the last edge into the final bin before the bounds check.
- [algorithmic-correctness, silent-failure] **GraphLayout.linear_point_to_bin_ind returns full-grid (gap-inclusive) indices, not the documented active-bin indices** — `src/neurospatial/layout/engines/graph.py:365-395`. With `edge_spacing>0` the returned index is shifted by preceding gap bins and can index out of range into the length-`n_active` `bin_centers`. Fix: remap full indices through `cumsum(active_mask)-1`, or correct the docstring and audit callers; add a gap test.
- [numerical-robustness, silent-failure] **Inconsistent occupancy threshold: regular/KDTree use `>` but hexagonal uses `>=`** — `src/neurospatial/layout/helpers/regular_grid.py:228` (also `utils.py:263` and `hexagonal.py:490`). Same `bin_count_threshold` silently yields different active masks (and occupancy/place fields) across layouts; factory docstring says "≥". Fix: pick one convention (docstring implies `>=`) and apply it uniformly; add a cross-engine equality test.
- [numerical-robustness] **Silent int32 overflow in get_n_bins for large extent / small bin_size** — `src/neurospatial/layout/helpers/utils.py:121`. `np.ceil(extent/bin_size).astype(np.int32)` saturates at 2147483647 with only a RuntimeWarning, silently producing a wrong grid. Fix: compute in int64 and raise a `ValueError` naming the dimension, extent, and bin_size when bins-per-axis exceeds a sane limit.
- [silent-failure] **_points_to_regular_grid_bin_ind returns all -1 on dimensionality mismatch instead of raising** — `src/neurospatial/layout/helpers/regular_grid.py:638-648`. A wrong-dim or transposed array warns and returns all -1, indistinguishable from "all points outside," so occupancy/decoding silently get zero valid samples. Fix: raise `ValueError` on shape mismatch (true out-of-bounds is already handled by digitize).
- [tests] **Public validators validate_bin_size and validate_dimension_ranges have zero test coverage** — `src/neurospatial/layout/validation.py:44-233`. Every error branch (NaN, inf, ≤0, inverted range, wrong n_dims, etc.) is untested and all doctests are `+SKIP`, so a regression could let invalid geometry through. Fix: add direct tests exercising each documented bad input and the happy-path returns.
- [tests] **N-D boundary-trimming branch of _infer_active_bins_from_regular_grid is untested** — `src/neurospatial/layout/helpers/regular_grid.py:266-273`. The only boundary test is 1D and hits the separate 1D branch; a bug in the multi-axis loop would silently leak boundary bins. Fix: add a 2D/3D test asserting the outer layer of every axis is False and the interior unchanged.
- [api-consistency, ux-ergonomics] **TriangularMeshLayout.build breaks keyword-only convention and skips @capture_build_params** — `src/neurospatial/layout/engines/triangular_mesh.py:77-89`. It is the only engine with positional params and stores `_build_params_used` under serialization keys (no `boundary_polygon`), so `get_layout_parameters` reports a different param kind and round-trip via `create_layout` raises "Unexpected arguments." Fix: make signature `(self, *, boundary_polygon, point_spacing)`, apply the decorator, and store serialized coords separately.
- [ux-ergonomics] **get_layout_parameters reports default=None for required args, hiding mandatory params** — `src/neurospatial/layout/factories.py:164-166`. Required `bin_size` and optional `dimension_ranges` both show `default: None`, defeating discovery. Fix: add a `required` boolean key via `param.default is inspect.Parameter.empty`.
- [ux-ergonomics] **create_layout "Unexpected arguments" error gives no valid-name hint** — `src/neurospatial/layout/factories.py:231-233`. A typo like `bin_sizes=2.0` reports the bad arg but not the valid ones (already computed on line 230). Fix: append `Valid arguments are: {sorted(allowed)}` (optionally `difflib.get_close_matches`).
- [docs-consistency] **environments.md documents nonexistent `close` param for from_samples (actual: `close_gaps`)** — `docs/user-guide/environments.md:43`. Copying the doc yields a swallowed/rejected kwarg. Fix: list `dilate`, `fill_holes`, `close_gaps`.
- [docs-consistency] **map_active_data_to_grid docstring omits required params grid_shape and active_mask** — `src/neurospatial/layout/helpers/utils.py:976-1009`. Documents only `active_bin_data`/`fill_value` and references `self.*` though it is a module-level function. Fix: document the two leading params and drop `self.` references.
- [performance-memory] **point_to_bin_index rebuilds the simplex lookup array on every call** — `src/neurospatial/layout/engines/triangular_mesh.py:215-219`. `orig2active_arr` is reconstructed via a Python dict loop on a hot path even though the map is fixed after build. Fix: precompute a dense int lookup array once at end of `build()` and index into it.

### Suggestions (nice to have)
- [numerical-robustness] **Zero-length graph edge causes 0/0 (NaN) in _project_1d_to_2d** — `src/neurospatial/layout/helpers/graph.py:408-415`. Coincident nodes (tolerated elsewhere) divide by zero edge length; `np.clip` does not repair NaN, which can poison bin_centers. Fix: set normalized position to 0 where edge length ≈ 0, or reject zero-length edges in build.
- [units-and-frames, docs-consistency] **_cartesian_to_fractional_cube docstring mislabels hex_radius as hexagon width** — `src/neurospatial/layout/helpers/hexagonal.py:182-203`. The code uses center-to-vertex distance but the docstring calls it "distance between opposite sides" and writes the denominator as `hex_width`. Fix: describe `hex_radius` as center-to-vertex (= hexagon_width/√3) and correct the formula text to `/ hex_radius`.
- [tests] **No-valid-positions warning path in _infer_active_bins_from_regular_grid is untested** — `src/neurospatial/layout/helpers/regular_grid.py:216-225`. Fix: add a test with all-NaN and empty position arrays asserting the UserWarning and an all-False mask of correct shape.
- [tests] **inverse_map fast-path of _points_to_regular_grid_bin_ind not tested against recompute path** — `src/neurospatial/layout/helpers/regular_grid.py:692-717`. A stale/off-by-one cached map would silently mismap bins. Fix: assert equivalence between `inverse_map=None` and a precomputed map across active/inactive/out-of-bounds points.
- [tests] **Hex point-to-bin correctness only verified for bin centers, not off-center points** — `tests/layout/test_hexagonal_layout.py:45-51`. The parity branch (`r_axial & 1`) is untested for off-center geometry. Fix: place points clearly inside known hexagons (odd and even rows) and assert expected flat indices.
- [silent-failure] **HexagonalLayout.point_to_bin_index returns all -1 with only a warning when no active bins** — `src/neurospatial/layout/engines/hexagonal.py:312-326`. Conflates "no bins" with "outside." Fix: raise `RuntimeError` when the layout has zero active bins; reserve -1 for genuine out-of-grid points.
- [api-consistency] **ImageMaskLayout.build param named bin_size but factory/docs use pixel_size** — `src/neurospatial/layout/engines/image_mask.py:62`. `create_layout('ImageMask', pixel_size=...)` raises "Unexpected arguments." Fix: rename to `pixel_size` (with `bin_size` deprecated alias) or document the mapping.
- [ux-ergonomics] **ImageMaskLayout.build defaults bin_size=1.0, silently assuming 1 unit/pixel** — `src/neurospatial/layout/engines/image_mask.py:58-63`. Contradicts `from_pixel_mask`'s required `pixel_size`. Fix: drop the default to force an explicit scale, or warn when omitted; document units as spatial-units-per-pixel.
- [docs-consistency] **layout-engines.md wrongly states MaskedGridLayout is auto-used when infer_active_bins=True** — `docs/user-guide/layout-engines.md:70-86`. from_samples only yields RegularGrid/Hexagonal; MaskedGrid comes via from_grid_mask. Fix: clarify RegularGrid handles inference in-place and MaskedGrid requires an explicit mask + grid_edges.
- [performance-memory] **point_to_bin_index boundary-containment filter uses a Python loop** — `src/neurospatial/layout/engines/triangular_mesh.py:239-242`. O(n_query_points) Python iteration in an otherwise vectorized routine. Fix: `active_triangle_idxs[idxs[~on_or_inside]] = -1`.
- [performance-memory] **Generic connectivity builder computes edge geometry one edge at a time in Python** — `src/neurospatial/layout/helpers/graph_building.py:151-188`. Millions of tiny per-edge NumPy/atan2 calls slow construction on fine grids. Fix: collect all edge candidates, then vectorize displacement/distance/angle over all edges (centers already available).
- [performance-memory] **Full dense grid of bin centers materialized even for very sparse active masks** — `src/neurospatial/layout/helpers/regular_grid.py:519-523`. Transient peak memory scales with total bins, not active bins (e.g. thin track in a large box). Fix: pass only active centers + a flat-index→active-index map to connectivity builders, or at minimum document the memory scaling.

### Strengths
- [statistical-validity] No statistical-validity issues surfaced — appropriate, as layout is geometric discretization rather than estimation.
- [reproducibility-rng] No RNG/reproducibility issues found; layout construction is deterministic with no hidden seeds.

## `regions`

### Critical (must fix before use)
- [silent-failure] **_rle_to_mask silently produces wrong mask on out-of-bounds or negative RLE indices** — `src/neurospatial/regions/io.py:212-250`. Docstring promises `ValueError` on out-of-bounds, but no check exists: out-of-bounds runs are silently truncated by NumPy slicing and negative starts write to the wrong (end) rows, yielding a plausible-but-wrong ROI feeding `_mask_to_polygon`. Validate each `(start, length)`: reject `start<0`, `length<0`, or `start+length>height*width`, and use `zip(..., strict=True)`.
