# Module Review: ["environment", "behavior", "events"]

## Executive Summary

**Findings by severity:** 13 critical, 42 important, 28 suggestion (83 total) across environment, behavior, and events.

**Top issues**

- [environment / algorithmic-correctness] **occupancy(time_allocation='linear') writes full-grid flat indices into the active-bin-indexed occupancy array, corrupting occupancy and losing mass on masked RegularGrids (the common `from_samples` case)** — `src/neurospatial/environment/trajectory.py:1116-1208,1312-1359`
- [behavior / numerical-robustness] **approach_rate divides by dt without guarding zero/duplicate timestamps → inf/nan poisons mean_approach_rate** — `src/neurospatial/behavior/navigation.py:1853-1857`
- [behavior / numerical-robustness] **mean_square_displacement: median dt of 0 (duplicate timestamps) causes division by zero in n_lags and degenerate linspace** — `src/neurospatial/behavior/trajectory.py:617-621`
- [behavior / silent-failure] **Out-of-bounds bins (-1) silently wrap to the last bin when indexing region masks/Voronoi labels, corrupting all region-based segmentation and decision analysis** — `src/neurospatial/behavior/segmentation.py:295,477-478,1313-1314` (also `decisions.py:767,904`)
- [environment / units-and-frames] **Polar egocentric env stores raw radians as edge 'distance' (cm/rad unit mixing), corrupting geodesic distances and density smoothing** — `src/neurospatial/environment/factories.py:980-1002`
- [events / silent-failure] **events_to_intervals(match_by=...) silently produces a Cartesian product when key values are non-unique** — `src/neurospatial/events/intervals.py:343-375`
- [events / numerical-robustness] **distance_to_reward silently produces wrong distances when reward_times contains NaN, and infers reward positions via np.interp assuming sorted `times`** — `src/neurospatial/events/regressors.py:617-783,666-672`
- [behavior / ux-ergonomics] **compute_vte_session / path_progress / cost_to_goal docstring + QUICKSTART examples are non-runnable (wrong arg order, arg collisions → TypeError)** — `src/neurospatial/behavior/vte.py:638-646`, `navigation.py:538-554,796-798`

**Cross-cutting patterns**

- **Active-bin vs. full-grid index confusion** on masked RegularGrids: the same class of bug recurs in environment occupancy (`trajectory.py`), `interpolate` (`fields.py:555`), and `rebin` diagonal inference (`transforms.py:258`). Any environment with inactive bins is mis-indexed or crashes.
- **Out-of-bounds bin (-1) wraparound**: `-1` indices silently read the last bin's mask/label across behavior segmentation and decisions (`segmentation.py`, `decisions.py:767,904`) — a defense-in-depth gap at the binning boundary.
- **Unguarded dt division**: zero/negative/duplicate timestamps poison rates and displacements throughout behavior (`navigation.py`, `trajectory.py`, `segmentation.py:533,673`).
- **`window` parameter semantics inconsistent** across events GLM regressors (scalar half-width vs tuple vs max_time); every documented `event_indicator` example (module docstring + QUICKSTART) passes a tuple and raises TypeError.
- **Non-runnable docstring/QUICKSTART examples** from wrong argument order recur heavily in behavior (path_progress, cost_to_goal, vte) and events.

**Recommended order**

1. **Correctness-of-results bugs that silently corrupt science first**: the environment active-bin/full-grid index mismatch (occupancy linear allocation + polar-egocentric unit mixing), the behavior `-1` bin wraparound in region/Voronoi indexing, and events `match_by` Cartesian product + `distance_to_reward` NaN/sort assumptions. These produce plausible-but-wrong numbers with no error.
2. **Numerical robustness guards**: add dt>0 and finite-timestamp validation across behavior rate/MSD/approach functions and environment trajectory timestamps — cheap, broad, prevents inf/nan propagation.
3. **API/doc breakage users hit immediately**: fix the non-runnable docstring/QUICKSTART examples and the `window` scalar-vs-tuple inconsistency, plus the stale `EnvironmentProtocol.animate_fields` stub.
4. **Tests last but tied to (1)**: add the missing inactive-bin occupancy and geodesic/approach-rate coverage that currently mask the correctness bugs above.

Detail for each item lives in the per-module sections below.

## `environment`

### Critical (must fix before use)
- [algorithmic-correctness][numerical-robustness][tests] **occupancy(time_allocation='linear') writes full-grid flat indices into the active-bin-indexed occupancy array** — `src/neurospatial/environment/trajectory.py:1116-1208, 1312-1359`. `_position_to_flat_index` returns a full-grid flat index but `_allocate_time_linear` accumulates into `occupancy` of length `n_bins` (active bins), so on any masked RegularGrid (the common `from_samples` case) time is silently dropped/misassigned to wrong physical bins, corrupting firing-rate denominators; map full-grid indices through `active_mask` to active-bin indices before accumulating, and add a regression test on a holed grid asserting mass conservation and correct bin placement (no `linear` test currently uses a non-dense grid).
- [units-and-frames] **Polar egocentric env stores raw radians as edge 'distance' (cm/rad unit mixing)** — `src/neurospatial/environment/factories.py:980-1002`. `from_grid_mask` sets non-wrap angular/diagonal edge distances to the Euclidean norm of `(distance_cm, angle_rad)`, so identical physical moves differ ~5x and geodesic distance/density smoothing are physically wrong on polar envs; recompute each edge distance as a proper polar length (arc length `r·Δθ`, radial `Δr`, diagonal `sqrt(Δr² + (r·Δθ)²)`) and fix `bin_sizes` accordingly, or block `metric='geodesic'` on polar envs.

## `behavior`

### Critical (must fix before use)

Duplicate timestamps yield inf, which `np.nanmean` (line 1922) propagates into `mean_approach_rate`; guard `dt > 0` (or validate strictly increasing times) and emit NaN.

## `events`

### Critical (must fix before use)
- [numerical-robustness/silent-failure] **distance_to_reward silently produces wrong distances when reward_times/positions contain NaN (no validation)** — `src/neurospatial/events/regressors.py:617-783`. Unlike sibling regressors, it never rejects NaN/Inf; a single NaN corrupts argsort/searchsorted ordering (NaN comparisons are False), flipping nearest-reward selection for valid samples — add the same NaN/Inf guards used by `time_to_nearest_event`.
- [silent-failure] **events_to_intervals(match_by=...) silently produces a Cartesian product when key values are non-unique** — `src/neurospatial/events/intervals.py:343-375`. The set-equality guard checks membership but not per-key uniqueness, so a repeated trial_id inner-merges into fabricated, plausible intervals — validate uniqueness in both frames (or merge with `validate='one_to_one'`) and raise.

### Important (should fix)
- [algorithmic-correctness/numerical-robustness] **distance_to_reward interpolates reward positions with np.interp on unsorted sample times** — `src/neurospatial/events/regressors.py:666-672`. `np.interp` requires monotonically increasing `times`; unsorted (e.g. session-concatenated) input silently returns wrong reward positions — sort `times`/`positions` together before interpolating, mirroring `add_positions`.
- [numerical-robustness/silent-failure] **add_positions silently returns all-NaN (or wrong) positions for degenerate trajectories** — `src/neurospatial/events/detection.py:141-162`. A single-sample trajectory makes interp1d divide-by-zero into all-NaN; NaN in trajectory `times` sorts to the end and corrupts the interpolant — validate `len(times) >= 2`, `np.ptp(times) > 0`, and finite `times`/`positions`.

### Strengths
- **reproducibility-rng**: production source in `src/neurospatial/events/` contains no randomness at all (confirmed by grep), so there are no reproducibility hazards in the shipping code — the only RNG nit is a test-fixture hygiene issue.
- **Temporal regressors are robustly validated**: `time_to_nearest_event`, `event_count_in_window`, and `event_indicator` all explicitly reject NaN/Inf inputs with clear WHAT/WHY/HOW messages, and `align_spikes_to_events` raises on NaN spike/event times with good test coverage — the gaps are confined to `distance_to_reward` and `add_positions`, which deviate from this otherwise consistent standard.
- **searchsorted-based temporal lookups** are used consistently across the module (the dense-broadcast and per-row-dict hot paths in `intervals`/`alignment` are the lone performance exceptions).
